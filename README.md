# 訪問看護 月次報告書 自動作成

カイポケからダウンロードした「1ヶ月分の記録PDF」をアップロードすると、月次報告書の次の3項目を自動作成します。

- **病状の経過**
- **看護リハビリテーションの内容**
- **家庭での介護の状況**

各項目は1〜5行程度にまとまります。作成済みかどうかの**重複チェック**（同一利用者・同一対象月）に対応しており、店舗内で報告書を共有できる仕様です。

---

## 手順（想定フロー）

1. **カイポケから対象利用者様の1ヶ月分の記録をPDFでダウンロードする**
2. **アプリを開き、PDFファイルをアップロードする**  
   - 利用者名・対象月を入力し、PDF（複数可）を選択して「報告書を生成する」をクリック
3. **およそ3分程度で報告書が生成される**（データ量により前後します）
4. **生成された内容を確認・修正する**  
   - 画面で3項目を編集し「修正を保存」可能
5. **カイポケの月次報告書画面に転記する**  
   - 「報告書全文をコピー」ボタンでクリップボードにコピーし、カイポケに貼り付け

処理中でも別のPDFをアップロード可能です。帰り際にまとめてアップロードしておけば、翌朝にはすべて生成できている想定です。  
生成した報告書は店舗内で共有されるため、リハ看護が入っている場合も二度手間にならずに利用できます。

---

## セットアップ

### 1. Python 環境

- Python 3.10 以上を推奨

```bash
cd 報告書自動作成
python -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 2. 要約のやり方（3通りから選べます）

| 方法 | .env の設定 | コスト | 備考 |
|------|-------------|--------|------|
| **AIなし（ルールベース）** | `SUMMARIZE_MODE=rule` | 無料 | 記録の見出し（■観察記録 等）から該当箇所を抜き出します。API不要でそのまま使えます。 |
| **Ollama（ローカル）** | `SUMMARIZE_MODE=ollama` | 無料 | Macに [Ollama](https://ollama.com) を入れ、`ollama run llama3.2` で起動。データは端末内のみ。 |
| **クラウドAPI** | `SUMMARIZE_MODE=api` ＋ APIキー | 無料〜月30円程度 | Groq（無料）／Gemini／OpenAI など。 |

**すぐ試すなら** プロジェクト直下の `.env` に `SUMMARIZE_MODE=rule` だけ書けば、APIキーなしで報告書が作成できます。必要ならあとから API や Ollama に切り替えてください。

**APIを使う場合** は `.env` に以下を追加します。

**OpenAI 本家を使う場合**

```env
OPENAI_API_KEY=sk-xxxxxxxx
# OPENAI_MODEL=gpt-4o-mini   # 省略時は gpt-4o-mini
```

**OpenAI 互換エンドポイント（自社サーバー等）を使う場合**

```env
OPENAI_API_KEY=your-api-key
OPENAI_BASE_URL=https://your-api-endpoint/v1
OPENAI_MODEL=your-model-name
```

### 3. 起動

プロジェクト直下で以下を実行します。

```bash
./run.sh
```

起動後、ターミナルに **このPCで開くURL** と **他の端末で開くURL** が表示されます。ブラウザで **http://127.0.0.1:8000** を開くか、表示されたURLを他の人に伝えて共有できます。

**他の端末から開けない場合**

- **同じWi-Fi（同じネットワーク）か確認**  
  スマホや別PCは、サーバーを動かしているPCと**同じWi-Fi**に接続してください。
- **表示されたIPアドレスを使う**  
  「他の端末で開く」に表示された `http://192.168.x.x:8000` のようなURLを、**そのまま**ブラウザのアドレス欄に入力してください。`localhost` は他の端末では開けません。
- **ファイアウォールでポートを許可する**  
  - **Mac**: システム設定 → ネットワーク → ファイアウォール → オプションで「uvicorn」または「Python」を許可するか、ポート 8000 を許可。  
  - **Windows**: コントロールパネル → Windows Defender ファイアウォール → 詳細設定 → 受信の規則で「ポート 8000」を許可。

### 4. URLを知っている人が誰でも開けるようにする（インターネット公開）

同じWi-Fiでなくても、**URLを知っている人ならどこからでも**開けるようにするには、無料のトンネルを使います。

**手順（2つのターミナルを使います）**

1. **1つ目のターミナル**でサーバーを起動  
   ```bash
   ./run.sh
   ```

2. **2つ目のターミナル**で公開用URLを取得  
   ```bash
   ./run-tunnel.sh
   ```  
   Node.js が入っていれば、`https://xxxx.loca.lt` のようなURLが表示されます。**このURLを共有すると、誰でもブラウザで開けます**（自宅・外出先・別のネットワークからも可）。

**Node.js がない場合**

- [Node.js](https://nodejs.org) をインストールしてから `./run-tunnel.sh` を実行するか、
- [ngrok](https://ngrok.com) に無料登録し、`ngrok http 8000` を実行して表示されたURLを共有してください。

※ トンネルは**サーバーを動かしている間だけ**有効です。サーバーを止めるとURLも使えなくなります。報告書データはPC内に保存されたままです。

### 常時インターネット公開（Railway）

**URLを常に有効にしたい**場合は [Railway](https://railway.app) にデプロイできます。手順は **[RAILWAY.md](RAILWAY.md)** を参照してください。

### 5. 自分のPC以外の端末で実装テストする

**方法A：同じWi-Fiのスマホ・別PCで開く**

1. 開発用PCで `./run.sh` を起動する。
2. ターミナルに表示される **「他の端末で開く: http://192.168.x.x:8000」** のURLをメモする。
3. テスト用のスマホや別PCを**同じWi-Fi**に接続し、ブラウザでそのURLを開く。

**方法B：別ネットワークや手軽に試す（トンネル）**

1. 開発用PCで **1つ目のターミナル** に `./run.sh` を起動する。
2. **2つ目のターミナル** で `./run-tunnel.sh` を実行する。
3. 表示された `https://xxxx.loca.lt` を、テスト用のスマホや別PCのブラウザで開く（Wi-Fiが違っても可）。

どちらでも「自分のPCじゃないもの」から同じアプリにアクセスして実装の確認ができます。

## 重複チェックについて

- **利用者名** と **対象月**（例: 2025-03）の組み合わせで重複を判定します。
- 既に同じ組み合わせの報告書がある場合、新規生成は行わず「既に作成されています」と表示し、既存の報告書を開くリンクを出します。
- 上書きしたい場合は、既存の報告書を開いて内容を修正するか、別の運用（対象月の表記を変える等）で対応してください。

---

## 注意事項

- PDFは**テキストが含まれるもの**を想定しています。スキャンされた画像のみのPDFの場合は、現バージョンではテキストを読み取れないことがあります（OCR対応は別途検討）。
- 報告書データは `backend/data/reports/`（または設定で変更したフォルダ）にJSONで保存されます。店舗内共有ではこのフォルダを共有ストレージに置くなどの運用が可能です。
