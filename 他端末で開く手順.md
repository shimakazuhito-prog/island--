# 他端末で報告書アプリを開く手順書

報告書自動作成アプリは、**サーバーを動かしているPC**と**開く側の端末**の2つが必要です。  
開く側は **Windows** や **iPad** など、ブラウザが使えれば利用できます。

---

## 前提

- **サーバー用PC**：アプリ（`./run.sh`）を起動しているMacまたはPC
- **開く側**：WindowsのPC、iPad、その他ブラウザが使える端末
- **ローカルネットワーク**：社内など同じネットワークでは、IPアドレスが **192.168.68.x** の形式になります。画面上部の「他端末で開く」に表示される**同じWi-Fi用**のURL（例：`http://192.168.68.10:8000`）で、同じWi-Fiの端末だけ開けます。
- **外から開く**：自宅・外出先など**別のネットワーク**から開くときは、IPのURLでは開けません。**方法2のトンネル**（`./run-tunnel.sh`）で表示される `https://xxxx.loca.lt` のURLを使ってください。

---

## 方法1：ローカルネットワーク（同じWi-Fi）から開く

同じオフィス・同じWi-Fiなら、**サーバー用PCのIP**で開けます。ローカルネットワークのIPは **192.168.68.x** のような形式です。

### サーバーを動かしているPC側

1. プロジェクトフォルダでターミナルを開く。
2. 次を実行する。
   ```bash
   ./run.sh
   ```
3. 表示された内容のうち、**「他の端末で開く:」**の行を確認する。
   - 例（ローカルネットワーク）：`他の端末で開く: http://192.168.68.10:8000`  
   - ※ 192.168.68.1 はルーターのことが多いため、表示されるのは 192.168.68.2 〜 254 のどれかです。
4. この **URL** をメモする。

### Windowsで開く場合

1. **同じWi-Fiに接続する**  
   - サーバーを動かしているPCと**同じ無線LAN（Wi-Fi）**に接続する。
2. **ブラウザを開く**  
   - Edge、Chrome、Firefox など。
3. **アドレス欄にURLを入力する**  
   - 例：`http://192.168.68.10:8000`
4. **Enterキーを押す**  
   - 報告書アプリの画面が表示されればOK。

### iPadで開く場合（ローカルネットワーク）

1. **同じWi-Fiに接続する**  
   - サーバーを動かしているPCと**同じWi-Fi**に接続する。
2. **Safari または Chrome を開く**  
   - iPad の Safari または Chrome アプリを起動する。
3. **アドレス欄にURLを入力する**  
   - サーバー側で表示されたURL（例：`http://192.168.68.10:8000`）を入力する。
4. **「開く」や「移動」をタップする**  
   - 報告書アプリの画面が表示されればOK。一覧・アップロード・編集・コピーなど、同じ機能が使えます。

**iPadで使いやすくする（任意）**

- **ホーム画面に追加**：Safari でアプリを開いた状態で、共有ボタン（□に↑）→「ホーム画面に追加」をタップすると、アイコンからすぐ開けます。
- **ブックマーク**：よく使うURLをブックマークしておくと便利です。

### 開けない場合（同じWi-Fiなのに）

- サーバー用PCの**ファイアウォール**でポート8000が許可されているか確認する。
- Windowsの**ファイアウォール**で「プライベートネットワーク」などでブラウザの通信が許可されているか確認する。
- URLの**数字（IP）と :8000** を間違えずに入力しているか確認する（`http://` も含める）。ローカルネットワークでは `http://192.168.68.xxx:8000` の形式です。

---

## よくある「開けない」とその対処（重要）

**「http://192.168.68.10:8000 で開けない」とき**

1. **IPアドレスが違う可能性**  
   `192.168.68.10` は例です。**サーバーを動かしているPCのIPは、そのPCで `./run.sh` を実行したときに表示される「他の端末で開く」のURLで確認してください。**  
   - ターミナルに `他の端末で開く: http://192.168.68.5:8000` と出ていれば、使うURLは `http://192.168.68.5:8000` です。  
   - `.10` ではなく、**表示された数字をそのまま**使ってください。

2. **サーバー用PCでまず確認**  
   サーバーを動かしているPCのブラウザで **http://127.0.0.1:8000** を開く。  
   - 開ける → サーバーは動いている。問題は「IP・ネットワーク・ファイアウォール」のどれか。  
   - 開けない → `./run.sh` が動いているか、エラーが出ていないか確認する。

3. **ファイアウォール（サーバー用PCがMacの場合）**  
   - システム設定 → ネットワーク → ファイアウォール → オプション  
   - 「Python」や「uvicorn」を「接続を許可」にするか、  
   - または「＋」で「ポート8000（TCP）」を追加して許可する。

4. **同じネットワークか**  
   開く側の端末（Windows・iPad）が、サーバー用PCと**同じWi-Fi（同じ192.168.68.xのネットワーク）**に接続されているか確認する。

5. **それでも開けないとき**  
   方法2の**トンネル**（`./run-tunnel.sh`）を使うと、ファイアウォールやIPを気にせず、表示された `https://xxxx.loca.lt` のURLでどこからでも開けます。

---

## 方法2：別ネットワークから開く（トンネル利用）

自宅や外出先など、**サーバー用PCと違うネットワーク**の端末（Windows・iPadなど）で開く場合です。

### サーバーを動かしているPC側

1. **1つ目のターミナル**で、次を実行してサーバーを起動する。
   ```bash
   ./run.sh
   ```
2. **2つ目のターミナル**（または新しいタブ）を開く。
3. 同じプロジェクトフォルダで、次を実行する。
   ```bash
   ./run-tunnel.sh
   ```
4. 表示された **https://xxxx.loca.lt** のようなURLをメモする。  
   - 例：`https://abc123.loca.lt`  
   - （Node.js が入っていない場合は、README の「Node.js がない場合」を参照して ngrok 等を利用）

### Windows側（他端末）

1. **ブラウザを開く**  
   - Edge、Chrome、Firefox など。
2. **アドレス欄にトンネルURLを入力する**  
   - 例：`https://abc123.loca.lt`
3. **Enterキーを押す**  
   - 初回だけ「Click to Continue」などと出ることがあります。表示に従ってクリックすると報告書アプリが開きます。

### iPadで開く場合（トンネル）

1. Safari または Chrome で、トンネルURL（例：`https://abc123.loca.lt`）をアドレス欄に入力する。
2. 初回だけ「Click to Continue」などと出たら、タップして進む。
3. 報告書アプリが表示されれば、ローカルネットワークのときと同じように使えます。

### 注意（トンネルの場合）

- トンネル用のターミナル（`./run-tunnel.sh`）とサーバー（`./run.sh`）の**両方を止めると、URLは使えなくなります**。
- 再度開くときは、もう一度 `./run.sh` と `./run-tunnel.sh` を実行し、表示された**新しいURL**をブラウザで開いてください。
- **任意**：トンネルで表示されたURLを `.env` に `PUBLIC_URL=https://xxxx.loca.lt` と書いてサーバーを再起動すると、画面上の「外から開く」欄にそのURLが表示され、コピーして共有しやすくなります。

---

## まとめ

| 項目 | 方法1（ローカルネットワーク） | 方法2（トンネル） |
|------|-------------------------------|-------------------|
| 接続 | サーバーPCと**同じWi-Fi**（192.168.68.x など） | どこでも可（インターネット接続があれば可） |
| 開くURLの例 | `http://192.168.68.10:8000` | `https://xxxx.loca.lt` |
| サーバー側でやること | `./run.sh` だけ | `./run.sh` と `./run-tunnel.sh` の2つ |
| 対応端末 | Windows、iPad、その他ブラウザ付き端末 | 同上 |

**他端末（Windows・iPadなど）で共通：**

1. ブラウザのアドレス欄に、サーバー側で案内されたURLを入力する。
2. Enter（または「開く」「移動」）で開く。
3. 報告書の一覧・アップロード・編集・全文コピーなど、同じ機能が使えます。

---

## トラブル時の確認

- **「このサイトにアクセスできません」**  
  - URLが正しいか（`http://` または `https://`、数字、`:8000` や `.loca.lt` まで）。  
  - 方法1のときは、同じWi-Fi（ローカルネットワーク）か。  
  - サーバー用PCで `./run.sh` が動いたままか。
- **「接続がタイムアウトしました」**  
  - サーバー用PCのファイアウォールで 8000 番が許可されているか。  
  - 方法1のときは、同じネットワーク（192.168.68.x など）か。
- **トンネルURLで「Click to Continue」が出る**  
  - 初回のみの案内なので、表示どおりタップ（クリック）して進む。
- **iPadで入力しづらい・ボタンが押しにくい**  
  - アプリはタッチ操作に対応しています。Safari の表示倍率を 100% にすると操作しやすい場合があります。

以上が、他端末（Windows・iPadなど）で開く手順です。
